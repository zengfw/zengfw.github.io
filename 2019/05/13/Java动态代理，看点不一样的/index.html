Java动态代理，看点不一样的

发表于 2019-05-13 更新于 2021-06-16 分类于 Java ， Android

了解Java动态代理之前，尝试花几分钟思考下面几个问题：

  • 静态代理和动态代理的区别是什么？
  • 动态代理类是怎么生成的？
  • 动态代理类代码结构是怎样的？
  • 动态代理具体的工作原理是什么？

一、静态代理

代理模式也称为委托模式，当无法或不想直接访问某个对象接口时，可以通过代理对象调用委托对象的接口，通常代理对象、委托对象需要实现相同的接口，代码示例…

二、动态代理

代理模式分静态代理和动态代理，静态代理如上代码示例，在编译时就知道了所有类的信息，但静态代理模式暴露的两个问题：

ISubject的子类必须实现所有接口；

ISubject新增接口时，对应子类不得不新增对应的接口实现

想解决上面的问题，是该考虑动态代理了。动态代理通过反射机制动态地生成代理对象，不需要实现接口，也不用担心ISubject接口陆续新增接口的问题了。代码示例Java动态代理的使用：

native层generateProxy方法，在程序运行时通过Runtime::Current()->GetHeap()在堆中动态生成class代理类文件，并根据java层传下来的参数，生成对应的成员方法、变量、构造函数等信息。

那生成的动态代理类代码结构又是怎样的呢？我们通过ProxyGenerator.generateProxyClass将动态代理类对应的class文件打印出来。

$Proxy0.class

动态代理类继承了Proxy类而持有InvocationHandler处理器，再重点看下subscribe()方法，InvocationHandler调用invoke方法，根据目标方法名、目标方法参数，间接调用委托对象的接口。

关于代理类名，比如$Proxy0，$Proxy是前缀，0是编号，多个代理类则依次递增。可参看上述Proxy.java源码中ProxyClassFactory在代理类名这块的处理。
