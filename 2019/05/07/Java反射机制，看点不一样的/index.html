一、类的信息 - Class对象

1. Java运行时识别对象和类信息的方式

主要有两种：

  • RTTI（Run-Time Type Identification），在编译时就知道了所有的类型、类的信息。
  • 反射，在运行时发现和使用类的信息。`

要想获取类的信息，自然少不了Class对象，它包含了与类有关的信息，每个类都有一个Class对象（.class文件：JVM通过类加载器生成）

2. Class对象的获取方式

  1. 通过已存在的对象，调用getClass()方法
  2. 根据目标类的文本名，使用Class.forName()
  3. 类字面常量获取，如People.class

这在一些业务场景还是比较有用的，举个比较粗糙的例子。

某菜单界面有多个item：A，B，C…，当后台配置了n个item时，假设程序会在某个时刻根据后台配置的item名，并通过Class.forName准备好菜单的所有Class对象信息时，这是菜单可能还不需要显示出来，我们则可以通过.class的方式，有效地实现了类对象初始化的“懒加载”。

通常项目大部分在编译时类的信息都是已知的，那如果类的信息从本地或者服务器下载下来的文件呢？对反射机制来说，.class文件是在运行时打开和检查的，对RTTI来说，.class文件是编译器在编译时打开和检查的。

反射的缺点

假设客户端通过反射使用某个依赖库里头多个私有接口方法，暂且命名为testHaha，testHehehe等等。哪一天依赖库版本更新了，你发现私有方法做了修改，名字由原先的testHaha，testHehehe变更为testHaha_V2，testHehehe_V2，这时你不得根据不同的版本对代码做兼容性处理。
